# Project Charter

## Project Title

University Course Registration System

## Project Sponsor

University Administration

## Project Manager

Project Management Intern (Case Study)

## Project Purpose

The purpose of this project is to design and propose a centralized digital platform that streamlines course registration processes for students, academic advisors, lecturers, and administrators.

## Business Need

The university currently relies on manual and semi-manual registration processes that cause delays, errors, and administrative inefficiencies. A digital solution is required to improve operational effectiveness and student satisfaction.

## Project Objectives

* Digitize course registration workflows.
* Reduce registration processing time.
* Improve data accuracy.
* Enhance student experience.
* Provide reporting capabilities for administrators.

## High-Level Deliverables

* Requirements Documentation
* Process Analysis
* UML Diagrams
* Risk Assessment
* Product Backlog
* AI Enhancement Proposal

## Success Criteria

* 80% reduction in registration processing time.
* Reduced registration errors.
* Improved student satisfaction.
* Efficient handling of peak registration periods.

## Constraints

* Limited project budget.
* Fixed academic semester deadlines.
* Compliance with university policies.

## Assumptions

* Students have access to internet-enabled devices.
* University authentication systems are available for integration.
* Stakeholders will participate in requirement validation.
