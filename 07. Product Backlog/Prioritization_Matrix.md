# Prioritization Matrix

## Prioritization Method

The MoSCoW prioritization framework was used to determine feature importance.

## Must Have

* User Authentication
* Course Registration
* Course Search and Viewing
* Timetable Generation
* Registration Approval Workflow

## Should Have

* Registration Status Notifications
* Enrollment Reporting
* Lecturer Class Lists

## Could Have

* Course Recommendations
* Chatbot Support
* Student Registration Analytics

## Won't Have (Current Release)

* Learning Management System Integration
* Tuition Payment Processing
* Attendance Tracking

## Rationale

Must-have features are required for the system to deliver its core business value. Should-have features improve operational efficiency, while could-have features enhance user experience and may be considered for future releases.
