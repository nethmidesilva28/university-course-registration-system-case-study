# Software Requirements Specification (SRS)

## System Overview

The University Course Registration System is a web-based application that supports student enrollment, advisor approvals, and course management.

## Functional Requirements

* User Authentication
* Course Search
* Course Registration
* Course Withdrawal
* Timetable Generation
* Approval Workflow
* Reporting

## Non-Functional Requirements

* 99% system availability
* Response time under 3 seconds
* Mobile responsive interface
* Secure authentication
* Role-based access control

## User Roles

### Student

Register and manage courses.

### Academic Advisor

Review and approve registrations.

### Lecturer

View enrolled students.

### Administrator

Manage courses and reports.

## System Constraints

* Integration with university authentication services.
* Compliance with university academic policies.
